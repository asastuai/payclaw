# LinkedIn Post

AI agents are getting smarter every month. They can research, plan, write code, manage tasks.

But the moment an agent needs to pay for something — an API call, a supplier, a refund — it stops and waits for a human.

I just open-sourced PayClaw: a developer SDK that gives AI agents wallets with programmable spending rules.

The difference from existing solutions? The rules are enforced on-chain by smart contracts, not in application code. Even if the agent is compromised, it can't exceed its limits. Think corporate card with spending rules vs. handing over your personal debit card.

What's included:
→ Smart contracts with daily limits, per-tx limits, token allowlists, and human approval flows
→ 24 tests including fuzz testing and invariant testing
→ Security audit with 7 findings identified and fixed
→ TypeScript SDK on npm (npm install payclaw-ai)
→ Multi-chain: Base L2, BSC, Solana

McKinsey projects $3-5T in agentic commerce by 2030. Google, Stripe, and Mastercard are building enterprise solutions for this. PayClaw is the open-source alternative for indie developers.
